# 本来为了省事用 qexo 编辑博客，实现无客户端，不用打代码，不用配置环境，直接 web 管理。

没想到啊没想到，居然让官方说明坑了一下，耗费两小时。

用 qexo 管理 hexo 博客，是挺方便的，使用图床的时候被坑两小时

说说经过吧，

# 本来设置图床有官方模板很简单，官方也做了详细说明

然后我就按官方说法一步一步配置，

嘿，不出意外的话出现了意外，

配置完测试上传图片居然报错！！！

😬 当时我就震惊了！！！！

# 错误信息：

```bash
JSONDecodeError('Expecting value: line 1 column 1 (char 0)')
```

我了个擦的，我返回设置页面，反复检查我的设置和官方有啥区别，

根本没检查出任何问题。

没办法只有去百度

结果为

读取普通 json 文件时出现此错误

此处应为 json 文件的格式错误

# 这就说明不是我填写错误，就是程序 bug

遂再仔细阅读官方文档，呵呵

```bash
{"Authorization": "abcdefg123456"}
```

仔细看 中间多个空格，冒号后多空格..........

# 最后去掉空格，测试成功，

**已联系作者，更改**

# 还有一个坑没填

就是文章更新时间，项目部署在 vercel，文章时间不是国内时间，看看能联系上作者不，等我问问吧，或可能自己想到了就改。

作者联系到了，说程序会自动变换时间 格林威治时间
